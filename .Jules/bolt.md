## 2025-04-18 - Avoid Dynamic Regex Recompilation with WeakMap
**Learning:** In high-frequency text replacement paths like `SecretScanner.redact`, recompiling config-driven regex objects (such as `config.customPatterns` or `config.whitelistPatterns`) repeatedly on every call causes severe performance degradation, doubling or tripling execution times. However, since the config arrays are often stable and passed by reference from external configuration (e.g. VS Code workspace config via `getConfig()`), they maintain referential equality during an execution pass.
**Action:** Use `WeakMap` to safely cache the heavily processed derived objects (like arrays of compiled `RegExp` objects) using the raw configuration array reference (e.g., `config.customPatterns`) as the WeakMap key. This avoids manual invalidation logic, prevents memory leaks since the cache clears when the config array is garbage collected, and provides immediate O(1) retrieval of the compiled regexes, drastically speeding up repetitive redact calls without breaking the stateless functional interface of the `redact` method.
