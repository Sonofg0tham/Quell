## 2025-04-03 - [CRITICAL] Fix Arbitrary Command Execution in Webview
**Vulnerability:** The webview message handler `onDidReceiveMessage` in `SidebarProvider.ts` accepted and executed any VS Code command passed to it via `data.command` without validation. If an attacker could inject an XSS payload into the webview, they could execute arbitrary VS Code commands (such as opening terminals or executing files) with the user's privileges.
**Learning:** Even if CSP is present, webviews often display data from untrusted sources (e.g., file paths, file contents). If the message handler blindly trusts the webview, an XSS vulnerability escalates into Remote Code Execution (RCE) / Arbitrary Command Execution within the editor.
**Prevention:** Always validate or whitelist command strings received from webviews before passing them to `vscode.commands.executeCommand`. In this case, ensuring `data.command.startsWith('quell.')` prevents the execution of built-in or other extensions' potentially dangerous commands.
