## 2024-05-24 - [Dangerous Webview HTML Interpolation Pattern]
**Vulnerability:** The webview UI used unescaped string interpolation to display secret findings, escaping only backslashes and single quotes for JS but omitting full HTML-escaping. This allowed potential Cross-Site Scripting (XSS) if a user opened a file with a maliciously crafted name.
**Learning:** Even though file paths generally appear safe, they represent untrusted user input that is reflected in the UI. VS Code Webview Content Security Policies that allow 'unsafe-inline' scripts (necessary for standard vscode.postMessage event handlers) are particularly vulnerable to this pattern. If a user downloaded a repository containing a file named `"><script>alert(1)</script>`, it could execute code within the Webview context. The specific codebase pattern of rendering HTML dynamically via `getHtmlForWebview()` makes this highly dangerous.
**Prevention:** Always rigorously HTML-escape all dynamic data inserted into HTML strings. When inserting data into inline event handlers, it must be properly JavaScript-escaped AND HTML-escaped. Avoid 'unsafe-inline' CSP if possible, but if required, enforce strict escaping at the point of interpolation.
