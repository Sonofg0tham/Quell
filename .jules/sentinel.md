## 2024-05-24 - Arbitrary Command Execution via Webview
**Vulnerability:** The VS Code webview `onDidReceiveMessage` handler accepted any command string and passed it directly to `vscode.commands.executeCommand(data.command)` without validation. If an attacker could inject malicious JavaScript into the webview (e.g., via a crafted file path or workspace finding that wasn't properly escaped, though escaping was present, defense-in-depth is required), they could execute arbitrary VS Code commands, including those from other installed extensions or built-in commands with severe side effects.
**Learning:** Webviews run in a separate context but can communicate with the extension host. Trusting the `command` string directly from the webview message payload allows for privilege escalation from the restricted webview context to the extension host context.
**Prevention:** Always strictly validate or whitelist the `command` string received from webviews before passing it to `vscode.commands.executeCommand()`. For example, verifying that the command starts with the extension's specific prefix (e.g., `quell.`).
