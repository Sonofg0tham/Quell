## 2024-05-24 - Parallelize I/O-bound tasks in workspace scans
**Learning:** Sequential `for` loops reading files block the event loop and increase latency for workspace scanning tasks. Since file reading and secret scanning don't depend on sequential execution order, they can be processed concurrently.
**Action:** Parallelize I/O-bound tasks in workspace scans (like file reading and secret scanning) using `Promise.all` with `.map()` instead of sequential `for` loops to significantly reduce execution time.
## 2024-05-24 - Optimize hot path string replacement and lookups
**Learning:** In high-frequency text replacement functions (like `SecretScanner.redact`), replacing O(N) linear iteration for lookups with O(1) reverse-lookup Maps (e.g., `valueToPlaceholder`) significantly improves execution speed. Also, for global string replacements in performance-critical paths, `String.prototype.replaceAll()` is faster and more memory-efficient than the `.split(delimiter).join(replacement)` pattern, as it avoids allocating intermediate arrays. When using `.replaceAll()`, if the replacement string is arbitrary or user-controlled, it must be passed as a callback function to prevent special replacement patterns (like `$`, `$&`, `$1`) from executing and corrupting the output.
**Action:** Use Maps for O(1) reverse-lookups in hot paths and prefer `String.prototype.replaceAll(..., () => replacement)` over `.split().join()` for global replacements to reduce memory allocations.
